<objective>
Replace the naive time.sleep(1) in main() with a deadline-based sleep helper that
aligns sentence emission to the UTC second boundary, preventing drift accumulation.

Purpose: The NixiChron clock's PPS trigger fires on the leading edge of '$'. For the
clock to display correct time, '$' must arrive as close to the UTC second boundary as
possible. A relative sleep(1) accumulates loop overhead as drift; a deadline sleep
self-corrects on every iteration.

      # Layer 5: Deadline-based timing (TIME-02, TIME-03)

      def sleep_until_next_second() -> None:
          """Sleep until the start of the next UTC second boundary.

          Deadline-based: computes next integer second from wall clock, sleeps the
          fractional remainder. Self-correcting per iteration — no drift accumulation.
          The max(0.0, ...) guard prevents ValueError if called at an exact second
          boundary where floating-point arithmetic produces a tiny negative value.
          """
          now = time.time()
          next_tick = math.ceil(now)
          time.sleep(max(0.0, next_tick - now))
